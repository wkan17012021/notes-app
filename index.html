PWA: Notes App

This is a note storage application for demonstration and testing only. It has limited progressive web enhancement (PWA). Built with Materialize CSS library, plain Javascript, Firestore noSQL database, Firebase BaaS.

How to use this app?

It has a basic login and logout system. As a first time user, sign up/register to access the app, provide a valid email address and at least a 6 character length password. After clicking the sign up button, you should be registered and logged in with your details ü§ûüèΩ

You should now be able to see / read all current notes. Please do not add sensitive or personal information. This app is for demonstration and testing only.

  • Click the orange plus button to create a new note.
  • Click the top-right icon in a note card to update a note.
  • Click the bottom-right icon in a note card to delete a note.

Log out using the button from the hamburger menu icon on the top-right navigation.

If viewing this app on a smartphone browser, you can simualte the "app installation", which adds this note-app URL link to your phone as an icon. More info in the 'About this App' section of the side menu.

Why did I create this app?

To build on what I was taught regarding a basic to-do list which was to use the web browser's local storage API. But the data only persists if you use the same device and that the local storage cache does not get flushed for any reason.

I wanted to use a to-do-list that has persisting data, can be accessed online, with a low backend overhead, which I can use conveniently e.g. not having to remember a shopping list of items while at the grocers.

What did I learn from the project?

  • To learn and apply some principles of progressive web enhancement (see the 'About this App' link in the sidemenu for more details)
  • Interpret documentation to work with a backend-as-a-service to hook up a noSQL database with my frontend.
  • Practice using plain JavaScript for DOM manipulation and UI updates.
  • How to pull together different tools to build a demo full stack application.

Anything unclear? Feedback?

There's certainly room for improvement: more secure authentication and authorisation; personalised accounts so each account should store their own specific notes; better UI styling, better handling of information and state client-side, to name a few.

If you have feedback, let me know by leaving a note!
